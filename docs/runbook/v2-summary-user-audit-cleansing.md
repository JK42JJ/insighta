# V2 Rich Summary — Per-User Audit & Cleansing Runbook

**Audience**: Internal admins only. Reads + targeted UPDATEs against prod
`video_rich_summaries` for a single user's rows.

**Created**: CP476 (2026-05-20). Source incident: user
`jamesjk4242@gmail.com` reported sidebar entries rendered as full
sentences instead of short mandala sub-goal labels. Measurement found
1 outlier row out of 80 across 16 mandalas.

**LLM API policy**: This runbook does NOT issue LLM calls. Labels are
either (a) untouched, (b) hand-written by the auditing admin, or (c)
left to be regenerated by the normal v2 cron path. Per CLAUDE.md
`§"LLM API 호출 금지"` this audit is a Hard-Rule-safe operation.

---

## 1 — Scope & Definitions

Audit target: `core.one_liner` and `analysis.core_argument` on
`video_rich_summaries.template_version = 'v2'` rows.

| Field | Used by | Spec |
|-------|---------|------|
| `core.one_liner` | Sidebar entry text (`SidebarLearningSection.tsx`) | ≤ 20 chars, mandala-sub-goal noun phrase, no trailing punctuation |
| `analysis.core_argument` | Grid card footer blockquote | 2-3 sentences, declarative, no "이 영상은…" filler |

Outlier categories (sidebar legibility):

- **L (length)**: `LENGTH(one_liner) > 20`
- **NK (non-Korean)**: `source_language != 'ko'` on a Korean mandala
  *(grid title in Korean but LLM returned English)*
- **S (sentence)**: matches `…ㅂ니다`, `…(이)다`, `…했어요`, `…합니다`
  — full sentence verbs that indicate the LLM ignored the noun-phrase
  rule

## 3 — Per-mandala distribution scan

Quick health check across every mandala the user has bookmarked. Look
for outlier max_len or non_ko.

```sql
WITH user_cards AS (
  SELECT video_id, mandala_id FROM user_local_cards
  WHERE user_id = '<USER_ID>'
  UNION
  SELECT yv.youtube_video_id AS video_id, uvs.mandala_id
  FROM user_video_states uvs
  JOIN youtube_videos yv ON yv.id = uvs.video_id
  WHERE uvs.user_id = '<USER_ID>'
    AND uvs.pinned_at IS NOT NULL
)
SELECT m.title AS mandala,
       COUNT(*) AS n,
       MIN(LENGTH(v.core->>'one_liner')) AS min_len,
       AVG(LENGTH(v.core->>'one_liner'))::int AS avg_len,
       MAX(LENGTH(v.core->>'one_liner')) AS max_len,
       SUM(CASE WHEN v.source_language IS DISTINCT FROM 'ko' THEN 1 ELSE 0 END) AS non_ko
FROM video_rich_summaries v
JOIN user_cards uc ON uc.video_id = v.video_id
JOIN user_mandalas m ON m.id = uc.mandala_id
WHERE v.template_version = 'v2' AND v.core IS NOT NULL
GROUP BY m.title
ORDER BY max_len DESC;
```

Healthy distribution (reference, observed 2026-05-20):
- max_len typically 9–17 chars
- non_ko = 0 for Korean mandalas

If a mandala shows `max_len > 25` or `non_ko > 0` → go to §4 for
per-row drill-down.

## 4 — Drill down to outlier rows in one mandala

Replace `<MANDALA_TITLE_LIKE>` with the title (or partial pattern with
`%`) of the suspect mandala.

```sql
WITH user_cards AS (
  SELECT video_id, mandala_id FROM user_local_cards
  WHERE user_id = '<USER_ID>'
  UNION
  SELECT yv.youtube_video_id, uvs.mandala_id
  FROM user_video_states uvs
  JOIN youtube_videos yv ON yv.id = uvs.video_id
  WHERE uvs.user_id = '<USER_ID>' AND uvs.pinned_at IS NOT NULL
)
SELECT v.video_id,
       v.core->>'one_liner' AS entry,
       LENGTH(v.core->>'one_liner') AS len,
       v.source_language AS lang,
       LEFT(y.title, 50) AS yt_title
FROM video_rich_summaries v
JOIN user_cards uc ON uc.video_id = v.video_id
JOIN user_mandalas m ON m.id = uc.mandala_id
JOIN youtube_videos y ON y.youtube_video_id = v.video_id
WHERE v.template_version = 'v2' AND v.core IS NOT NULL
  AND m.title LIKE '<MANDALA_TITLE_LIKE>'
ORDER BY LENGTH(v.core->>'one_liner') DESC
LIMIT 30;
```

## 5 — Hand-write replacement labels

For each outlier row, the auditing admin writes a replacement label.
**Do not call an LLM.** Use the YouTube title and the mandala center
goal as input, target the same noun-phrase shape as healthy entries
(e.g. "월배당 ETF 전략", "노후 자산 관리").

Replacement heuristic:
1. Read the YouTube title and the surrounding healthy entries in the
   same mandala for tonal consistency.
2. Compose a noun phrase ≤ 18 chars (leaves 2-char headroom under the
   sidebar cap).
3. No trailing punctuation. No "이 영상은…". No comma-enumerated lists.
4. Stay in the user's mandala language (Korean for Korean mandalas).

Examples observed CP476:

| YouTube title (truncated) | Hand-written label | Chars |
|---------------------------|--------------------|-------|
| "방구석에서 노트북 하나로 6억 버는 요즘 미국 10대들…" | 미국 10대 부자 사례 | 12 |

## 6 — Apply the UPDATE

```sql
UPDATE video_rich_summaries
SET core = jsonb_set(core, '{one_liner}', to_jsonb('<NEW_LABEL>'::text))
WHERE video_id = '<VIDEO_ID>'
  AND template_version = 'v2';
-- expect: UPDATE 1
```

Verify by re-running §4 → confirm the row's `entry` is the new label.

The user must hard-refresh (`Cmd+Shift+R`) to clear FE cache; the
sidebar will re-render the corrected entry on the next refetch (PR #708
5s safety-net poll picks it up automatically within 5 seconds).

## 7 — Defer-to-cron alternative (no manual label)

If the admin does not want to author labels by hand, mark the row's
`quality_flag = 'low'` and let the cron Track A2 retry it on the next
12h tick (PR #693). The retry runs under the strengthened CP476+
prompt + 20-char hard cap (PR #712) so the new label should be
compliant.

```sql
UPDATE video_rich_summaries
SET quality_flag = 'low',
    updated_at = NOW() - INTERVAL '13 hours'  -- ensures Track A2 picks it up next tick
WHERE video_id = '<VIDEO_ID>'
  AND template_version = 'v2';
```

Trade-off: this re-runs the full LLM pipeline (Haiku + Sonnet). Cost
≈ $0.005 per row. Use when there are many outliers and hand-writing
labels is impractical.

## 8 — Code guards (already shipped)

- **PR #712** (CP476+) — `trimOneLinerLabel(raw)` soft-truncates the
  validator output at 20 chars and strips trailing punctuation. New
  rows generated after PR #712 deploy never exceed 20 chars at the
  sidebar layer, even if the LLM returns a longer string.
- **PR #712** — prompt rule strengthened with explicit GOOD/BAD
  examples; reduces (but does not eliminate) LLM rule violations.
- **Test**: `tests/unit/skills/rich-summary-v2-quick-prompt.test.ts`
  (9 cases).

Older rows generated before PR #712 deploy are NOT retroactively
trimmed; the FE renders them as-is. Use §6 (hand-write) or §7
(defer-to-cron) to clean them up.
