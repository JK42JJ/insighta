# Insighta — 나를 조금씩 키워가는 곳

> **Status**: Long-term vision (장기 과제)
> **Owner**: JK (Founder)
> **Date**: 2026-03-19
> **Priority**: Strategic — all feature decisions must align with this vision
> **Scope**: This document defines the WHY behind Insighta. Every technical decision, feature implementation, and architectural choice should be evaluated against these principles.

## 1. Core Philosophy (핵심 철학)

### 시작점

나는 게으르고 부족한 사람이다. 그런데 조금씩이라도 나아지고 싶다. Insighta는 그런 나를 다그치지 않고, 옆에서 같이 가자고 하는 것이다. 강약 조절은 필요하지만, 본질은 **내가 나를 독려하며 잘 다독이는 것**이다.

### One-Line Vision

**"디지털에서 자신의 신뢰를 키워가는 플랫폼"**

거창하게 들릴 수 있지만 실체는 단순하다 — 내가 뭔가 하고 싶은 걸 정하고, 관련 영상을 보고, 메모하고, 다시 보고... 이 과정을 Insighta가 기록해 주고, 내가 얼마나 왔는지 보여주고, 비슷한 사람과 자연스럽게 연결해 주는 것이다.

### Essence (본질)

> **"내가 성장하기 위한 실천을 돕는 것."**

이것이 Insighta가 하는 딱 하나의 일이다. 소통은 그 성장을 더 크고 정확하게 만들어주는 자연스러운 확장이지 별도의 기능이 아니다. 혼자 공부하면 100만큼 성장한다면, 같은 목표를 가진 사람의 그래프를 보면서 하면 150이 되는 것이고, 그건 당연한 일이다.

**"실천 → 기록 → 성장 → 연결 → 더 큰 성장"** — 이 하나의 흐름이 자연스럽게 돌아가게 만드는 것이 Insighta의 전부다. 사용자는 그냥 자기가 하고 싶은 걸 하면 되고, 나머지는 Insighta가 알아서 연결해 준다.

### "어쩌면 게임이야" (Founder's Insight)

Insighta는 게임과 같다. 게임에서 캐릭터를 키우는 이유는 남한테 보여주려고가 아니라 **내가 강해지니까**. 레벨 올리고 스킬 찍고 새로운 던전 깨는 게 재밌으니까. 근데 그 과정에서 자연스럽게 다른 플레이어가 내 캐릭터를 보고 "이 사람 진짜다"를 알게 된다.

기존 gamification(Duolingo식 연속 출석 뱃지, XP)은 **외부 보상**으로 동기를 만든다 — 보상이 사라지면 동기도 사라진다. Insighta는 **성장 자체가 보상**이다. 내가 진짜로 아는 게 늘고, 그래프가 풍부해지고, "나 이만큼 왔구나"를 느끼는 것. 이건 외부 보상 없이도 지속되는 동기다.

잘 못하는 부분이 보이는 것도 강점이다. 게임에서 약한 스탯을 보면 "여기를 키워야겠다"는 동기가 생긴다. 만다라트의 빈 영역도 마찬가지 — 부끄러운 게 아니라 **다음 목표가 보이는 것**이다.

### Positioning: Obsidian + Social Learning

Obsidian 사용자들은 지식 구조화에 진심이다. 제텔카스텐, 링크드 노트, 그래프 뷰... 하지만 그것이 **내 볼트 안에서 끝난다.** 혼자 쌓고 혼자 보는 것이다.

아이러니하게도, 이런 사람들이야말로 소통하면 시너지가 가장 크다. 체계적으로 정리된 지식을 가진 사람끼리 연결되면 서로의 blind spot을 발견할 수 있다. "나는 이 개념을 이렇게 연결했는데, 너는 저쪽으로 연결했구나" — 이 순간이 진짜 학습이다.

정리 잘 하는 사람들이 소통을 안 하는 게 아니라, **소통할 수 있는 구조가 없는 것이다.** Insighta는 이 간극을 채운다:

- **Obsidian**: 나의 두 번째 뇌 (개인 지식 관리)
- **Insighta**: 나의 두 번째 뇌가 **다른 사람의 두 번째 뇌와 만나는 곳** (지식 관리 + 소셜 학습)

### Problem Statement

현재 디지털 세상에서 신뢰를 표현하는 방식은 모두 피상적이다:

- 팔로워 수, 좋아요 수, 뱃지, 인증 마크 → 숫자이거나 외부에서 부여된 라벨
- 행동과 분리되어 있고, 살 수도 있고, 조작할 수도 있음
- 온라인 관계는 맥락이 부재하여 현실에서 만나면 서먹함
- 익명성 뒤에 숨은 폭력이 만연함

현실에서 사람을 신뢰하게 되는 과정은 다르다 — 그 사람이 **꾸준히 행동하는 것을 주변이 관찰**하면서 자연스럽게 형성된다. Insighta는 이 현실의 신뢰 구축 방식을 디지털에서 구현한다.

### Core Belief

> 사람 사이의 불신과 오해를 기술로 조금이라도 줄일 수 있다.
> 인생은 길고 복잡하고, 서로 오해가 생기고 미워하기도 한다.
> 이러한 문제를 아주 약간이라도 해소하는 것이 Insighta의 존재 이유다.

### Natural Noise Filtering (자연적 노이즈 필터링)

유튜브, SNS 등의 근본적 문제 — 광고, 오류, 폭력, 클릭베이트, 자극적 콘텐츠 등 노이즈가 너무 많다. 알고리즘이 "더 오래 머물게 하는 것"을 추천하기 때문에, 사용자의 의지와 상관없이 노이즈에 노출되는 구조다.

**Insighta는 이 노이즈가 구조적으로 걸러진다.** 사용자가 의도적으로 저장하고, 메모하고, 반복 재생하는 콘텐츠만 Knowledge Graph에 들어가기 때문이다. 옵시디언 하는 사람이 광고, 폭력, 노이즈를 모으지는 않는 이치와 같다.

이것은 **별도의 필터링 알고리즘이 필요 없다**는 뜻이다:
- 유튜브: AI 모더레이터, 신고 시스템, 정책 변경으로 유해 콘텐츠를 걸러내려 노력하지만 불완전
- Insighta: **사용자의 의도적 행동 자체가 최고의 필터** — 학습하려는 진지한 의도로 저장/정리한 콘텐츠만 남음

페르소나 구독과 결합되면 더 강력해진다 — 내가 구독하는 사람도 같은 방식으로 의도적으로 큐레이션한 콘텐츠만 공유하므로, 그 사람을 통해 들어오는 정보도 이미 필터링된 상태다. 노이즈가 끼어들 구조 자체가 없다.

## 2. Persona Model (페르소나 모델)

### Concept: 만다라트 = 페르소나

각 만다라트는 단순한 "목표"가 아니라 **사용자의 한 측면(페르소나)**을 나타낸다. 여러 페르소나가 모여 전체 "나"를 이룬다.

**Example — JK의 페르소나들:**

| 만다라트 | 페르소나 | 소비 패턴 |
|---------|---------|----------|
| A | 개발자 JK | 시스템 설계, 아키텍처, 오픈소스 |
| B | 창업자 JK | 제품 전략, 비즈니스 모델, 시장 분석 |
| C | 영어 학습자 JK | 영어 강의, 원어민 표현, TED |
| D | 자기계발하는 JK | Atomic Habits, 네트워킹, 커뮤니케이션 |

### Cross-Persona Value

하나의 콘텐츠가 여러 페르소나에 동시에 연결될 수 있다:
- "영어로 진행되는 스타트업 피칭 영상" → 창업자 JK + 영어 학습자 JK
- Knowledge Graph에서 페르소나 간 공유 노드 = **정체성의 교차점**
- "이 개념은 개발자인 당신과 창업자인 당신 모두에게 중요합니다" 같은 인사이트 가능

### Technical Mapping

```
Persona = Mandala (existing entity)
Persona Interests = Nodes connected to Mandala's sub-goals
Cross-Persona Node = Node with edges to multiple Mandalas
Identity Intersection = Shared concepts across personas
```

## 3. Trust Through Action (행동으로 증명하는 신뢰)

### Principle: 그래프는 거짓말을 못 한다

유튜브 연동은 시작점일 뿐이다. 진짜 Knowledge Graph를 만드는 것은 **그 이후의 행동**이다:

| 행동 | 의미 | 데이터 포인트 |
|------|------|-------------|
| 영상 저장 | 관심 표현 (낮은 신호) | resource node 생성 |
| 특정 구간 반복 재생 | 그 개념이 어렵거나 중요함 | segment replay count |
| 메모 작성 | 능동적 학습 참여 | note node + edge |
| 일주일 후 재방문 | 주제를 놓지 않고 있음 | revisit timestamp |
| 관련 자료 추가 연결 | 지식 구조화 시도 | cross-resource edges |
| 인사이트 도출 | 독자적 해석/종합 | insight node |

이 행동들은 **숨길 수 없다.** 프로필은 꾸밀 수 있고 자기소개는 포장할 수 있지만, "6개월간 시스템 설계 영상 47개를 보면서 메모 120개를 남기고, 특정 개념을 5번 이상 재방문한 기록"은 만들어낼 수 없다.

### Time Axis (시간축의 중요성)

시간축이 더해지면 **진정성의 깊이**가 드러난다:

- "AI 공부합니다"라고 프로필에 쓰는 것 ≠ 6개월간 매주 AI 콘텐츠를 소비하고 정리한 그래프
- 일시적 관심 vs 장기적 진지한 몰입을 구분할 수 있음
- 사람의 **생각의 변화 궤적**이 기록됨 → 특정 시기에 어떤 관심사를 가졌는지 추적 가능

### Immutable Record (변경 불가능한 기록 — 블록체인 연동)

> **장기 과제**: 시간축 위의 행동 기록은 변경할 수 없어야 한다.

- 블록체인 기술을 활용하여 학습 행동 기록의 무결성(integrity)을 보장
- "나는 3년 전부터 이걸 공부해왔어" → 블록체인에 기록이 있으면 증명이 필요 없음
- 기록 자체가 증거가 되는 구조
- 오해가 생겼을 때, 그 시기의 행동 궤적을 통해 맥락을 이해할 수 있음

**Implementation considerations:**
- 전체 그래프를 on-chain에 올릴 필요 없음 → 해시값만 주기적으로 앵커링
- 사용자가 원하는 기록만 선택적으로 공증(notarize)하는 방식 가능
- 초기에는 timestamp + hash로 시작, 향후 블록체인 통합

## 4. Persona Subscription (페르소나 구독)

### Concept: 지식 구조의 공유

사람은 교류를 통해 점차 넓은 경험, 기술 등으로 성장한다. Insighta의 페르소나 구독은 이 현실의 성장 방식을 디지털에서 구현한다.

**기존 SNS 구독:**
- 콘텐츠 피드를 받음 (그 사람이 뭘 올렸는지)
- 피상적 — 맥락 없는 정보 소비

**Insighta 페르소나 구독:**
- 그 사람의 **지식 그래프의 일부**를 받음
- 어떤 개념들을 연결했는지, 어떤 순서로 학습했는지, 어떤 인사이트를 도출했는지
- **콘텐츠가 아니라 사고의 구조**를 공유하는 것

### Trust-Based Connection

**구독 전제: 거부감 없는, 어느 정도 믿을 수 있는 사람**

이것은 트위터처럼 수천 명을 팔로우하는 모델이 아니다:
- 소수의 신뢰할 수 있는 사람과 깊게 연결
- 상대방의 지식 그래프를 보고 학습 궤적을 이해한 상태에서 구독
- 연결 자체에 이미 맥락과 존중이 내포됨

**지식 그래프가 보여주는 신뢰의 증거:**
- 특정 개념 노드에 연결된 리소스 수, 메모 수, 방문 빈도, 시간 투자량
- 별도로 "관심도를 입력하세요"를 요구할 필요 없음
- **사용자의 행동이 곧 데이터** (수동적 소비 시각화 철학과 일치)

### Information Propagation (정보 전파의 효율)

**현재의 정보 공유:**
A가 좋은 영상 발견 → 카톡에 링크 공유 → B가 읽었는지 모름 → C는 놓침 → 일주일 뒤 "그거 봤어?"

**Insighta에서의 정보 전파:**
A가 영상 저장 → AI 요약/핵심 개념 자동 추출 → 같은 페르소나 구독자 B, C의 그래프에 자동 연결

핵심 차이:
- 별도의 공유 행위 없이, **학습하는 것 자체가 전파**
- 콘텐츠가 아니라 **해석과 인사이트가 흐름**
- **비동기적이면서 자동** — 시간대, 일정, 응답 여부에 의존하지 않음
- 공동의 학습 목표를 가진 그룹에서는 만나고 전화하고 메시지 보내는 과정보다 빠름

### Anti-Toxicity Effect (SNS 폭력 방지 효과)

Insighta의 구조는 기존 SNS의 익명성 뒤에 숨은 폭력을 구조적으로 억제한다:

1. **행동 가시성**: 내가 무엇에 관심 있고 얼마나 오래 노력해왔는지가 그래프에 보임
2. **맥락 충돌**: 악의적 행동이 자신의 학습 그래프 맥락과 모순됨
3. **깊이 전제**: 구독 구조 자체가 피상적 연결이 아닌 깊은 이해를 전제
4. **시간축 검증**: 일시적 관심인지 장기적 진지함인지 시간이 증명

## 5. Trust Virtuous Cycle (신뢰의 선순환)

```
꾸준한 학습
  → Knowledge Graph 성장
    → 진정성이 시간축 위에 드러남
      → 페르소나 구독이 형성됨
        → 서로의 인사이트가 비동기적으로 흐름
          → 더 좋은 학습이 촉진됨
            → Graph가 더 풍부해짐
              → 신뢰가 깊어짐
                → 온라인/오프라인 경계 없는 관계로 발전
```

### Why Real-World Meetings Won't Be Awkward

일반적으로 온라인에서 만난 사람을 오프라인에서 만나면 어색한 이유:
- 서로에 대해 아는 것 = 닉네임, 프로필 사진, 몇 마디 댓글
- 실체 없는 관계

Insighta에서 서로의 그래프를 구독하고 있었다면:
- 만나기 전에 이미 상대가 뭘 진지하게 파고들고 있는지 알고 있음
- 어디서 막혀 있는지, 어떤 인사이트를 도출했는지도 파악 가능
- 이것은 현실에서 6개월 같이 공부한 스터디 멤버 수준의 맥락
- 처음 만나도 "그 영상에서 그 부분, 저도 여러 번 봤어요"로 대화 시작 가능

## 6. Value Chain (가치 사슬)

전체 가치 흐름을 단계별로 정리:

```
[Stage 1] 목표 설정
  만다라트(페르소나) 생성 — "나의 이 측면을 성장시키겠다"

[Stage 2] 콘텐츠 소비
  유튜브 연동 — 수동적 소비가 자동으로 기록됨

[Stage 3] 깊은 학습 행동
  반복 재생, 메모, 재방문, 자료 연결 — 행동이 그래프에 기록됨

[Stage 4] AI 요약 & Knowledge Graph
  Qwen3.5-9B 등 멀티 모델로 요약 — 품질 점수로 관리
  개념, 키워드, 인사이트가 노드/엣지로 자동 구조화

[Stage 5] 패턴 시각화
  소비 패턴 대시보드 — 시간대별, 감정별, 주제별
  "거울을 비추는" 수동적 시각화 → 자기 인식 촉진

[Stage 6] 그래프가 진정성을 증명
  시간축 위의 변경 불가능한 기록 (블록체인 앵커링)
  행동의 궤적이 곧 신뢰의 증거

[Stage 7] 페르소나 구독
  신뢰할 수 있는 사람과 지식 구조 공유
  정보 전파 효율 극대화 — 학습이 곧 공유

[Stage 8] 관계의 깊이
  온/오프라인 경계 없는 맥락 있는 관계
  SNS 폭력 구조적 억제
  인생의 오해와 불신을 조금이라도 줄이는 것
```

## 7. Persona Character System (페르소나 캐릭터 — 3D 인터페이스)

### Concept: Knowledge Graph의 인격화

Knowledge Graph는 데이터를 관리하는 **엔진**이다. 3D 캐릭터는 그 데이터를 사용자에게 전달하는 **인터페이스**다. 사람은 차트보다 캐릭터에 감정적으로 반응한다.

### 다마고찌 모델

각 페르소나(만다라트)마다 고유한 3D 캐릭터를 생성한다. 이 캐릭터는 사용자의 학습 행동에 따라 성장하고 변화한다:

- 꾸준히 학습하면 → 캐릭터가 성장하고 활기차짐
- 학습이 뜸하면 → 캐릭터가 조용해짐 (다그치지 않음, 사실만 전달)
- 특정 영역에 집중하면 → 캐릭터의 외형/특성이 그 방향으로 진화
- 페르소나마다 다른 외형, 말투, 성장 경로 → 각 페르소나에 대한 애착 형성

### Communication (캐릭터가 전달하는 정보)

**실시간 (웹 접속 시):**
- Welcome 인사: "오늘 구독한 분이 흥미로운 자료를 추가했어요"
- 학습 제안: "어제 반복 재생한 데코레이터 부분, 관련 자료 하나 찾았어요"
- 인사이트 공유: "이번 주 Python 관련 영상 5개를 보셨어요"

**비동기 (미접속 시, 다음 날 오전 메시지):**
- 채널: Telegram, Slack, KakaoTalk, WhatsApp 등 (사용자 설정)
- 톤: 친근하게, 사실 기반, 절대 다그치지 않음
- 예시: "어제 못 들어오셨네요. 구독한 그래프에 새로운 인사이트가 2개 추가됐어요 🙂"

**주간/월간 리포트:**
- 캐릭터가 전달하는 형태의 상세 리포트
- 시각화 포함 (관심 영역 변화, 학습 패턴, 성장 궤적)
- 사실과 행동에 기반 — 판단이나 압박 없이
- 리포트 상세도는 사용자 설정에서 조절 가능

### Character Design Principles

| 원칙 | 설명 |
|------|------|
| **사실 기반** | 캐릭터의 모든 발화는 실제 학습 행동 데이터에 기반 |
| **다그치지 않음** | "안 하셨네요" 대신 "이번 주는 조용했어요. 충전 중이신가요?" |
| **개인화** | 각 페르소나(만다라트)마다 고유한 캐릭터 |
| **성장 연동** | 학습 행동에 따라 캐릭터 외형/레벨 변화 |
| **감정적 연결** | 사용자가 캐릭터에 애착을 느끼도록 설계 |

### Technical Considerations

- 3D 캐릭터 생성: Three.js 또는 WebGL 기반 경량 렌더링
- 캐릭터 커스터마이징: 초기 선택 + 행동 기반 자동 진화
- 메시지 생성: LLM (Qwen3.5-9B)이 학습 데이터 기반으로 자연어 생성
- 외부 메시징: Telegram Bot API, Slack Webhook, KakaoTalk 알림톡 등
- 리포트 생성: 주간/월간 자동 생성 (크론 작업) + 사용자 설정에 따른 상세도 조절

## 8. Business Model Implications (비즈니스 모델 방향)

이 비전에서 자연스럽게 파생되는 수익 모델:

### 7-1. Knowledge Graph Subscription (지식 그래프 구독)
- 전문가가 자기 페르소나를 유료 공개
- "10년차 백엔드 개발자의 시스템 설계 지식 그래프, 월 $5"
- 강의도 뉴스레터도 아닌, **구조화된 전문 지식의 구독**이라는 새로운 카테고리
- 별도 콘텐츠 생산 불필요 — 평소 학습/정리 자체가 상품

### 7-2. Premium AI Features
- 멀티 모델 요약 (Standard/Premium 모델 선택)
- 고급 패턴 분석 (감정 레이어, 장기 트렌드)
- 페르소나 간 교차 인사이트

### 7-3. Group Learning
- 팀/스터디 그룹 단위 구독
- 공동 학습 목표에 대한 진행 상황 공유
- 조직 내 지식 전파 효율화

## 9. Feature Decision Framework

새 기능을 평가할 때 이 질문으로 판단:

> **"이 기능이 사용자가 디지털에서 신뢰를 키워가는 데 도움이 되는가?"**

### Alignment Check:
- [ ] 사용자의 행동이 자연스럽게 기록되는가? (추가 노력 요구 최소화)
- [ ] 시간축 위에서 의미 있는 패턴을 보여주는가?
- [ ] 진정성을 숨길 수 없는 구조인가?
- [ ] 다른 사용자와의 신뢰 기반 연결을 촉진하는가?
- [ ] 기존 SNS의 피상적 관계와 차별화되는가?

### Anti-Patterns (이런 기능은 만들지 않는다):
- 좋아요/팔로워 수 기반 인기도 시스템
- 익명 상호작용
- 바이럴/engagement 최적화 기능
- 행동 없이 획득할 수 있는 뱃지/등급
- 원본 콘텐츠를 그대로 재배포하는 기능

## 10. Technical Requirements for This Vision

### 9-1. Knowledge Graph (기반 인프라 — 현재 진행 중)
- ontology schema: nodes, edges, embeddings (PostgreSQL + pgvector) ✅
- service ≠ system domain separation ✅
- cross-domain edge validation 🔜
- MA-2: Knowledge Graph View 🔜

### 9-2. AI Summary Quality System (품질 관리 — 단기)
- 멀티 모델 동시 호출 + 품질 점수 비교
- 모델별/콘텐츠 유형별 최적 라우팅 데이터 축적
- 사용자 설정에서 모델 선택 (Free / Standard / Premium / Custom API Key)
- AWS Spot (us-west-2 Oregon) + OpenRouter API 병행
- 일관된 품질이 최우선 — 비용보다 데이터 품질이 크리티컬

### 9-3. Consumption Pattern Tracking (소비 패턴 — 중기)
- 시간대별 소비 히트맵
- 감정 레이어 (콘텐츠 무드 분류)
- 특정 구간 반복 재생 추적
- 재방문 패턴 기록

### 9-4. Persona & Subscription System (페르소나 구독 — 장기)
- 만다라트를 페르소나로 확장
- 페르소나별 visibility 설정 (private / subscribers / public)
- cross-user knowledge graph edge (`SUBSCRIBES_TO`)
- 구독자에게 인사이트 비동기 전파
- 유료 페르소나 구독 결제 (토스페이먼츠 + 포트원)

### 9-5. Immutable Record (변경 불가능 기록 — 장기)
- 학습 행동의 timestamp + content hash 생성
- 주기적 블록체인 앵커링 (해시값만 on-chain)
- 선택적 공증(notarize) 기능
- 기록 무결성 검증 API

### 10-6. Persona Character System (3D 캐릭터 — 장기)
- 페르소나별 고유 3D 캐릭터 생성 (Three.js / WebGL)
- 학습 행동 기반 캐릭터 성장/진화 로직
- LLM 기반 캐릭터 발화 생성 (사실 기반, 친근한 톤)
- 외부 메시징 연동 (Telegram, Slack, KakaoTalk, WhatsApp)
- 주간/월간 자동 리포트 생성 파이프라인
- 사용자 설정: 알림 채널, 빈도, 리포트 상세도
