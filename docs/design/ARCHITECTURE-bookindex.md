# Insighta 아키텍처 SSOT — 북인덱스 중심 파이프라인

> **이 문서의 목적**: "v2 → 완성된 북인덱스(책) → 무엇이든 생성" 구조를 단일 진실원천으로 고정한다.
> **상태**: 정렬용 아키텍처 SSOT (구현 핸드오프 아님). 코드/DDL 0건.
> **거주지**: in-repo. 권장 경로 `docs/design/ARCHITECTURE-bookindex.md` — 메모리 의존은 망각을 보장하므로 반드시 레포에 박는다.
> **계보**: 이 구조는 새 개념이 아니다. 노트모드 북인덱스(CP447), hub-and-spoke master, 6/22 "완성데이터=북인덱스" 재정의에서 반복적으로 합의됐다. 본 문서는 그 합의 + 2026-06-23 영상-우선 구조 실측 + 8셀 뼈대/게이트 명료화를 하나로 합쳐 고정한 것이다.

## §0 한 줄 정의

**완성데이터 = 만다라 단위로 완성된 북인덱스(책)다.** v2가 재료, 북인덱스가 SSOT, 슬라이드덱은 그 책에서 파생되는 여러 산출물 중 하나일 뿐이다.

```
v2 (재료) → 북인덱스(완성된 책, SSOT) → ├ 슬라이드덱
                                          ├ 뉴스레터 / 리포트
                                          ├ 챗봇 컨텍스트
                                          ├ 지식그래프(KG)
                                          └ ... (책의 fan-out)
```

세션 내내 슬라이드만 보며 땜빵한 것이 잘못이었다 — **책을 안 보고 책의 한 출력만 고치려 한 것.** 책이 완성되면 슬라이드는 책의 충실한 투영이 된다. 책이 raw면 어떤 출력을 뽑아도 raw다.

## §2 북인덱스가 게이트다 (핵심 원리)

게이트는 별도 단계가 아니라 **북인덱스 그 자체**다.

- **뼈대 = 8개 셀.** 만다라니까 뼈대는 셀 구조다. 셀 라벨(주제군 8개)은 이미 깨끗하며, 이것이 "이 책이 담아야 할 내용"의 목차 골격이다.
- **배치 = 내용차원으로 재구성.** 영상이 어느 셀에 박혔느냐가 아니라, 내용이 어떻게 이어져야 하느냐로 순서/구성이 정해진다. 영상 경계를 넘어 내용 단위로 재배치돼도 된다. 순서는 내용차원으로 변경 가능.
- **선별 = 인덱스에 기여하는 카드만.** v2가 게이트가 아니다. 북인덱스가 "필요한 내용"을 정의하고, 그 기준으로 카드를 거른다. rel=5(예: "기술주 투자"가 자동화 만다라에), 저조회수 조악 콘텐츠는 인덱스 구성 단계에서 **배제** — 포함시키면 평균 품질이 떨어진다.
- **v2는 통과분에만.** v2를 다 만들어 놓고 쓰는 게 아니라, 인덱스에 기여할 카드만 v2 비용을 쓴다. (워밍/CV도 동일 — 슬라이드 안 될 영상까지 numerize = 낭비.)

이것이 VISION 원칙과 정합한다 — **"필터는 개선이 아니라 게이트여야 한다."** 단일 카드 시절엔 사람이 게이트였고(입력을 손으로 큐레이션), 만다라 자동화는 그 게이트를 시스템에 이식하는 것이다. 게이트 없이 양만 늘리면(만다라=많은 셀) 쓰레기도 비례해 늘어난다.

## §3 현재 상태 = 영상-우선 (2026-06-23 실측) — "내용→스냅샷"은 0% 미착공

코드/DB 측정으로 확정: 목표 구조("내용이 주체, 스냅샷/figure는 내용에 종속")는 **코드에 0% 구현**돼 있고, 전 사슬이 영상-우선이다.

| 단계 | 현재 (영상-우선) | 근거 (file:line / 테이블) |
|---|---|---|
| 배치 | 영상이 셀에 박힘 | `user_video_states.cell_index` |
| book 생성 | placed 영상 열거 → 섹션 = 영상 | `fill-book.ts:68-75` "Enumerate placed videos per cell" |
| 섹션 정체성 | `section.title = video.title` (영상 원제목) | `build-book.ts:128` (내용 라벨 아님) |
| figure 연결 | **book에 figure/media 필드 0건** | `build-book.ts` — 섹션이 figure를 참조/요구 안 함 |
| 워밍 | 영상 ts(고관련 구간) → 프레임 긁기 | `warm-select` → `get-or-extract(videoId, ts)` |
| collect | placed 영상 video_id로 snapshots 수집 | `collect-figures.ts` — 영상-키 평행 수집 |

**핵심**: book과 figure가 내용으로 연결되지 않는다. figure는 영상 기준으로 따로 모아 빌드 요청에 별첨될 뿐 — "내용이 스냅샷을 주도"가 아니라 "영상 프레임을 긁어 옆에 붙임"이다. content-first 인덱스(내용 항목을 영상과 독립 정의하고 그게 수집을 주도) = 0건.

## §4 결함은 하나의 뿌리 — "영상-우선"

2026-06-22~23 덱 실물 검수에서 나온 결함들은 별개 버그가 아니라 **영상-우선 단일 뿌리의 증상**이다. 그래서 슬라이드젠을 증상별로 땜빵하면 매번 다음 증상이 나온다.

| 결함 | 증상 | 뿌리 |
|---|---|---|
| 1 섹션 제목 = 유튜브 원제목 (이모지·채널·URL·해시태그·`(N/M)`) | 발표 제목이 클릭베이트 | 섹션 = 영상 (`build-book.ts:128`) |
| 2 무관 영상 셀 배치 (rel=5 placed) | 주식 영상이 자동화 만다라에 | 배치에 관련도 게이트 없음 |
| 3 figure 0 | CV 데모인데 시각 오브젝트 없음 | book에 figure 슬롯 0 + 영상-키 평행 수집 |
| 5 얇은 덱 | v2usable만 섹션화 → 1~2섹션 | 영상 우선 + v2 인리치 안 됨 |

→ 북인덱스 중심 구조(§1)로 전환하면 1·2·3·5가 뿌리에서 동시에 사라진다. (섹션=내용이지 영상 아님 / 게이트가 조악 배제 / 책이 figure 타겟 보유 / 완성도 레이어가 균형 확보.)

> **함정 기록 (2026-06-23)**: 데모로 빌드한 "AI 에이전트로 업무 자동화하기"(42706d64, 7섹션 thin)는 의도한 rich 942e2757(61섹션)이 **아니다.** 같은 제목 다른 만다라 = 동명 충돌. 빌드 전 `mandala_id`로 rich본 확정 필수.

## §5 figure 재배치 — distill 대상 아님, render-time on-demand

| v1.2 (폐기) | 재정의 (확정) |
|---|---|
| 완성데이터 = `video_figures` (영상 단위 figure 누적) | 완성데이터 = **완성된 북인덱스** (만다라 단위 문서) |
| CV를 데이터완성 시점에 전수 전진배치 | CV는 **slidegen이 관련도 높은 구간에 요청하는 on-demand** |
| distill-once 대상 = figure | distill-once 대상 = **북인덱스(텍스트)**. figure는 render-time 파생 |
| 선행 = figure 스키마 | 선행 = **북인덱스 스키마** (figure는 북인덱스·관련도 위에서만 의미) |

figure/스냅샷은 완성된 책을 입력으로, 책의 figure 타겟이 "이 내용을 보여줄 시각자료"를 주도해 수집한다. 비용을 위해 on-demand + 캐시. (figure-rank 프레임 선택 수정은 배포됨 — 영상에 figure가 있으면 잡고, 없으면 정직하게 0. 수율은 이제 코드가 아니라 콘텐츠가 정한다.)

## §6 레포 경계

- **경계 = 완성된 북인덱스.** insighta는 영상을 완성된 책으로 끌어올리는 데까지 책임. slidegen은 그 완성 책을 받아 시각화하는 데까지 책임.
- insighta는 slidegen이 받을 SSOT(북인덱스 + 구간 관련도)를 생산. slidegen은 그 책을 직접 수정하지 않는다(0파일 원칙 / book_json = SSOT).
- 이 문서가 REPO_BOUNDARIES.md의 미결 항목(James 경계 입력 대기)을 채운다.

## §8 불변 원칙 (이 구조가 흔들릴 때 돌아올 지점)

1. **완성데이터 = 북인덱스(책), not figure, not 영상 요약.**
2. **북인덱스가 게이트.** 기여 카드만 v2 — 전수 v2/전수 워밍 아님.
3. **뼈대 = 8셀, 배치 = 내용차원.** 섹션은 내용 단위지 영상 원제목 아님.
4. **책이 figure를 주도.** figure는 render-time on-demand 파생, distill 대상 아님.
5. **경계 = 완성된 북인덱스.** insighta 생산 / slidegen 소비.
6. **슬라이드는 fan-out의 하나.** 책을 고치지 슬라이드를 땜빵하지 않는다.
