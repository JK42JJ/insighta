## §1. 핵심 / 본질

사용자 인용 (card-refresh-strategy.md § 1 인용 직접 보존):

> "카드는 한번에 최대한 많이 가져와야 해. 그중 일부를 24~30 등으로 기준에
> 의해 보여주고, 사용자가 추가 refresh 를 하는 경우 **나머지 카드 + 신규
> 카드 + 캐시 등으로 다양한 카드를 추가로 제공**할 수 있어야 함."

> "본인이 찾는 카드라면 like 를 누르거나. 그 다음부터는 카드 추천할 때
> 해당 카드의 내용을 바탕으로 **가중치를 일부 조정**. 그러나 유튜브처럼
> 도배하면 안 되. **일부 가중치만 조금 더** 주는 거야."

### 1줄 본질

사용자가 "Add Cards" 를 누르면 **(a) 나머지/신규 다른 set 을 (b) mandala
주제 안에서 (c) channel/sub_goal caps 로 도배 차단 (d) like signal 을
multiplier 로만 (≤20%) 반영 (e) progressive 도착 + score-desc 정렬** —
도배 없이 다양성 유지하며 큐레이션을 점진적으로 사용자에 맞춰 정제.

### 본질 → mental model

- **흡수** (handoff doc Phase 5 인용): "발견한 영상이 내 큐레이션으로 흡수"
- Add (= curate-to-mandala) ≠ Refresh (= passive re-fetch). 명칭 반영.

## §2. 사용자 시나리오 (확정)

```
[Grid view 우상단]
  "+ Add Cards"  chip 클릭                        ← trigger
         ↓ (~250ms slide-in)
[Right panel 열림, 폭 ~45% (desktop), 모바일 v2 보류]
  🔒 [center_goal 표시 — readonly]                  ← locked base (서버 resolve)
  🏷  [+ 키워드]  [×초보자]  [×실전]              ← chip 추가 (자동완성)
  ─────────────────────────────────
  [40 cards, score desc, progressive 도착]
  ┌──────────────┐ ┌──────────────┐ ...
  │ ☐ thumbnail │ │ ☐ thumbnail │
  │   title     │ │   title     │
  │       🔖 🗑 │ │       🔖 🗑 │
  └──────────────┘ └──────────────┘

  Single add: 🔖 hover-click
            → 카드가 왼쪽으로 ~400ms animation
            → grid 빈 셀 (best-cosine cell) 또는 scratchpad fallback
            → POST /like (signal + v2 enrich trigger)
            → user_video_states INSERT + surfaced_at=now()
            → 메인 grid 의 기존 Pick UI (Issue #649 Phase 3, CP466 Bookmark icon swap) 재사용

  Bulk add:  ☐ 체크 → panel 하단 sticky [N cards 추가]
            → batch animation
            → scratchpad 일괄 stage (cell 선택 popup 은 v2+)
            → 각 카드 user_video_states INSERT + surfaced_at=now()
```

**메인 grid 왼쪽 always visible** — 흡수 효과 즉시 보임 (handoff doc 인용).

## §4. Schema (확정)

```sql
ALTER TABLE public.user_video_states
  ADD COLUMN IF NOT EXISTS surfaced_at TIMESTAMPTZ;

CREATE INDEX IF NOT EXISTS idx_user_video_states_surfaced
  ON public.user_video_states (user_id, mandala_id, surfaced_at DESC)
  WHERE surfaced_at IS NOT NULL;
```

- `surfaced_at` NULL = never surfaced via Add Cards panel.
- 갱신 시점: 사용자 Pick 클릭 또는 bulk 추가 → user_video_states INSERT
  또는 UPDATE 시 `surfaced_at = now()`. Cells-side auto-add pipeline
  (`auto-add-recommendations.ts`) 는 `surfaced_at` 건드리지 않음 (별
  surfacing 의미 — 본 컬럼은 사용자 explicit "Add" 만).

기존 LIVE schema 재사용:
- `card_interactions` (Issue #649 CP462+) — `signal='like'/'archive'/'delete'`.
  Add Cards = `like` signal 발화 (단일 add) 또는 explicit add path (bulk).
- `recommendation_cache` (rec_score / cell_index / mandala_id) — backlog.
- `youtube_videos` + `video_pool_embeddings` (cosine match).

## §5. BE — `POST /api/v1/mandalas/:mandalaId/add-cards`

### Request body

```ts
{
  extraKeywords: string[];   // chip 키워드 (자동완성: subjects + signal + history)
  excludeVideoIds: string[]; // 이미 mandala 안 / archived / deleted signal videoId
  filters?: {                // CP466 amendment v1: 3 facets (post-filter, in-memory)
    minViewCount?: number;        // 1_000 / 10_000 / 100_000 / 1_000_000 presets
    durationBucket?: 'short' | 'medium' | 'long' | 'xlong';
                                  // short=<10m, medium=10-30m, long=30-60m, xlong=≥60m
    publishedAfter?: string;      // ISO date — '7d' / '30d' / '180d' / '365d' presets at FE
  };
}
```

### Filter semantics

- **`minViewCount`**: keep candidates with `viewCount >= minViewCount`. NULL viewCount falls through (kept). Defaults: panel preset = no filter (all).
- **`durationBucket`**: keep candidates within bucket bounds (seconds):
  `short` ≤ 600, `medium` 600–1800, `long` 1800–3600, `xlong` ≥ 3600.
  NULL durationSec falls through (kept).
- **`publishedAfter`**: keep candidates with `publishedAt >= publishedAfter`. NULL publishedAt falls through (kept). FE sends ISO string computed from preset days.

### Trigger semantics (search button)

- Panel **first open** → 1 auto-fetch on mount (initial backlog).
- Subsequent: **chip add/remove + filter change** disables auto-refetch.
  Explicit "Search" button click triggers fetch with current keywords + filters.
- Spec: docs/design/add-cards-2026-05-18.md §2 (updated CP466 amendment).

### Response

```ts
{
  status: 'ok',
  data: {
    cards: AddCardCandidate[],   // 최대 40 cards, score desc 정렬
    trace?: {                    // ?trace=1 query param 일 때만
      layer1_count: number,
      layer4_boost_applied: number,
      caps_enforced: { channel: number, subgoal: number },
      drift_guard_fired: boolean
    }
  }
}
```

`AddCardCandidate`:
```ts
{
  videoId: string,        // YouTube 11-char id
  title: string,
  channel: string | null,
  thumbnail: string | null,
  durationSec: number | null,
  viewCount: number | null,
  publishedAt: string | null,
  score: number,          // final composite (Layer 1-4 합성)
  cellIndex: number,      // best-cosine cell (auto-assign on Pick click)
  source: 'cache' | 'video_pool' | 'realtime'
}
```

### Internal pipeline (확정)

```
1. authenticate (Supabase JWT)
2. resolve center_goal from mandalaId (SELECT user_mandala_levels WHERE depth=0)
3. resolve sub_goals (depth=0.subjects[])
4. embed center_goal + extraKeywords → 1 batch (Mac Mini Ollama)
5. fetch candidates:
   - Tier 1: video_pool_embeddings cosine vs center_emb (LIMIT V3_TIER1_KNN_LIMIT)
   - Tier 2: existing user_video_states.surfaced_at IS NULL OR < 24h ago (Layer 1 Coverage)
   - exclude: excludeVideoIds (request) + user_local_cards (user-added) +
              user_video_states (already in mandala) +
              card_interactions WHERE signal='delete' (user-global block) +
              card_interactions WHERE signal='archive' AND mandala_id=current
6. apply mandala-filter (semantic gate, V3_CENTER_GATE_MODE=semantic)
7. assign best-cosine cell (max cosine over sub_goals[])
8. Layer 4 Feedback bias multiplier:
   - load card_interactions WHERE user_id=current AND signal='like' (LIMIT 100 recent)
   - compute liked_centroid (time-decayed avg embedding)
   - drift guard: if cosine(liked_centroid, center_emb) < 0.5 → Layer 4 disable
   - else: candidate score *= (1 + min(α_channel * channel_match + α_embed * cos(candidate, liked_centroid), 0.20))
9. caps enforcement (post-rank):
   - channel ≤ 2 / response — replace overflow with next-best from pool
   - sub_goal ≤ 4 / response (= 50% of 8 cells)
10. sort score desc + cellIndex asc tiebreaker
11. take 40
12. emit response (no surfaced_at write — write happens on add path, not surface path)
```

### Add path (separate write)

Add Cards panel UI 가 Pick (Bookmark) 클릭하면 **기존 `POST /api/v1/cards/:videoId/like`**
(LIVE, `cards.ts:280`) 호출. 본 PR 에서 같은 endpoint 의 INSERT path 에
`surfaced_at=now()` 갱신 추가 — `user_video_states` UPSERT 시:

```sql
INSERT INTO user_video_states (user_id, video_id, mandala_id, cell_index,
                                level_id, is_in_ideation, auto_added,
                                surfaced_at, pinned_at, created_at, updated_at)
VALUES (..., now(), now(), now(), now())
ON CONFLICT (user_id, video_id) DO UPDATE
  SET surfaced_at = now(),
      pinned_at = EXCLUDED.pinned_at,
      mandala_id = EXCLUDED.mandala_id;
```

### Auth + RLS

- Bearer JWT (Supabase session) — 기존 `fastify.authenticate` decorator.
- 모든 query 가 `user_id = current` scope.

### Error codes

- 401 UNAUTHORIZED
- 404 MANDALA_NOT_FOUND (mandalaId 가 user 소유 아님)
- 400 INVALID_BODY (extraKeywords / excludeVideoIds 형식 위반)
- 500 ADD_CARDS_FAILED (내부 에러)

### Smoke test

`tests/smoke/add-cards.test.ts`:
- POST /add-cards 401 (no token)
- POST /add-cards 404 (other user mandala)
- POST /add-cards 200 (own mandala, empty body)
- POST /add-cards 200 with extraKeywords + excludeVideoIds

## §6. FE — `frontend/src/widgets/add-cards-panel/`

### 파일 구성

```
frontend/src/widgets/add-cards-panel/
├── ui/
│   ├── AddCardsPanel.tsx           # main slide-in panel
│   ├── AddCardsTriggerChip.tsx     # grid 우상단 chip
│   ├── KeywordChipInput.tsx        # chip 입력 (자동완성)
│   ├── AddCardsList.tsx            # 40 candidates grid
│   └── AddCardsBulkBar.tsx         # sticky bottom "N cards 추가"
└── model/
    ├── useAddCards.ts              # TanStack mutation → POST /add-cards
    ├── useKeywordSuggest.ts        # chip 자동완성 (subjects + signal + history)
    └── useAddCardsPanelStore.ts    # Zustand (open/close/selectedIds)
```

### 행동 spec

| 행동 | 트리거 | 결과 |
|---|---|---|
| Panel open | `+ Add Cards` chip 클릭 | slide-in 250ms ease-out, fetch initial 40 |
| chip 추가 | input enter / autosuggest 클릭 | debounce 300ms → useAddCards refetch |
| chip 삭제 | × 클릭 | 즉시 refetch |
| Pick single | 카드 🔖 (단일 `Bookmark` icon + fill toggle: outline = unsaved hover, filled = idle saved with scale-[0.65]) | `POST /:videoId/like` (mandalaId 포함, signal=like enum 유지) + cross-panel animation 400ms + 카드 grid 빈 셀 또는 scratchpad |
| 체크 multi | ☐ → ☑ | selectedIds Set update + bulk bar 노출 |
| Bulk add | sticky bar `[N cards 추가]` | 각 selectedId 에 `POST /:videoId/like` 일괄 호출 + batch animation + scratchpad stage |
| Card click | thumbnail | 메인 `VideoPlayerModal` 재사용 (panel 위 modal) |
| Panel close | Esc / 외부 클릭 / × 버튼 | slide-out 250ms |

### Locked base UI

```tsx
<div className="px-4 py-3 border-b">
  <div className="flex items-center gap-2 text-sm text-muted-foreground">
    <Lock className="h-3.5 w-3.5" />
    <span>{centerGoal}</span>  {/* readonly, 서버에서 받은 값 */}
  </div>
</div>
```

### State management

- Zustand `useAddCardsPanelStore`: `{ open, selectedIds, extraKeywords }` —
  panel close-then-reopen 시 state 복원 (handoff doc Phase 5 spec).
- TanStack mutation 결과는 staleTime 60s 로 cache (rapid open/close 시
  re-fetch 비용 절감).
