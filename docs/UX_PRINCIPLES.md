# Insighta UX 원칙 (SSOT)

> 제정: 2026-06-11 (James, CP499+ — 자동추가 언어 진단에서 단편 전환 차단용으로 선언).
> 지위: **모든 기능 PR 은 본 문서와의 정합을 PR 본문에 1줄 명시할 의무**가 있다.
> 정합하지 않으면 구현 전에 보고한다. 위치 주석: 원 지시는 `docs/design/` 이었으나
> 해당 디렉토리는 gitignored (CP497 확인) — tracked SSOT 로서 docs 루트에 둔다.

## 원칙 1 — 언어 일관성: 입력 언어 = 경험 언어

- 사용자가 목표를 입력한 언어가 그 만다라의 경험 언어다.
- **자동 유입 (위저드 자동추가 / 카드 더 찾기 기본) = ko 만다라면 ko only.**
  영어-우세 제목은 제외한다. 한국어+영어 용어 혼합 제목 (한글 ≥1자) 은
  한국 영상이므로 유지한다.
- **English 칩 = 명시 opt-in.** 켜면 EN only (섞기 없음 — replace-mode, #895).
- v2 요약 표시 = 만다라 언어 (영어 영상의 v2 는 영어로 생성하되 표시는
  번역본 — 정확성과 일관성의 분리).
- 언어 판정 = **한글 우선** (#902 `detectLanguage`: 한글 1자라도 있으면 ko;
  고유명사는 언어 신호가 아니다).

## 원칙 2 — 콘텐츠 품질: 자동 유입은 세 관문을 통과한 것만

- **관련 (relevant)**: R1 3축 루브릭 점수 + A-2 게이트 캐시
  (`video_mandala_relevance`) 를 통과한 후보만 자동 유입.
- **시의성 (timely)**: volatile 도메인만 recency 우대 (70/30 쿼터 — 숫자는
  설정값, 법칙 아님). evergreen 은 무적용 — 오래된 요가 영상은 결함이 아니다.
- **생존 (alive)**: 삭제/비공개 영상은 prune (videos.list `status` 는 이미
  수집 중 — 추가 quota 0).
- **빈 셀 = 정직한 부족.** 무관 콘텐츠로 채우지 않는다. 빈 셀의 충당 수단은
  풀 서빙 (한국어 풀, A-2 게이트 위) 이다.

## 원칙 3 — 통제권: 자동은 보수적으로, 명시는 요청 그대로

- 시스템이 알아서 넣는 것 (자동추가) 은 보수적 기준 — 의심스러우면 제외.
- 사용자가 명시적으로 요청한 것 (English 칩, 수동 추가, 검색) 은 요청
  그대로 — 시스템이 임의로 걸러내지 않는다.

## 충돌 해소 기록

| 충돌 | 해소 (결정 주체: James) |
|------|------|
| **ko-only ↔ 빈 셀** (K8s/Docker 류 영어-압도 주제는 ko-only 시 빈 셀 증가) | 풀 서빙이 짝 — 두 기능은 **한 묶음으로 취급**. "무관 영어 (신뢰 파괴)" 제거가 "빈 셀 (정직한 부족)" 보다 우선이므로 언어 fix 가 선행될 수 있으나, 풀 서빙이 즉시 후속해야 하며 장기 공백 금지. |
| **70/30 ↔ evergreen** (일괄 recency 쿼터는 evergreen 을 부당 강등) | volatility 분기 — volatile 만 적용, evergreen·미분류(NULL) 무적용 (가산 방식, 감점 금지). |
| **EN 칩 ↔ v2 언어** (EN 칩으로 들어온 영어 영상의 요약 언어) | 원칙 1 우선 — v2 생성은 영상 원어 (정확성), **표시는 만다라 언어** (번역본). EN 칩은 검색 유입만 바꾸고 표시 언어 규칙을 바꾸지 않는다. |

## 적용 절차

1. 기능 PR 본문에 "UX 원칙 정합: 원칙 N — <1줄>" 명시.
2. 정합 불가/모호 시 구현 전 보고 (원칙 수정이 필요하면 본 문서 PR 로).
3. 본 문서의 숫자 (70/30 등) 는 전부 설정값 — 게이트 측정으로 재조정한다.
